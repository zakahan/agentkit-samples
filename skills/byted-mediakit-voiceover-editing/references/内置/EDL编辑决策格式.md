# EDL 编辑决策格式

> 口播剪辑的片段级编辑描述，供 Step 5 产出及导出使用。**时间单位统一为 ms**。

## 字段说明

| 字段        | 类型   | 必填 | 说明                                            |
| ----------- | ------ | ---- | ----------------------------------------------- |
| text        | string | ✓    | 处理后文本（concat 时为拼接结果）；**delete 时填 source_text**，便于展示被删除内容 |
| source_text | string | ✓    | 原始 ASR 文本                                   |
| start_time  | number | ✓    | 该片段在源音视频中的起始时间（ms）；**concat 删前留后时须为原句起始**（含被删部分），不可写成保留部分起始 |
| end_time    | number | ✓    | 该片段在源音视频中的结束时间（ms）              |
| reason      | string | ✓    | 处理原因，需明确                                |
| action      | string | ✓    | 处理动作：`mute` / `delete` / `concat` / `keep` |
| actionTime  | array  | ✓    | 处理时间范围，见下表                            |

## action 与 actionTime

| action     | 含义                                               | actionTime 格式                                                    |
| ---------- | -------------------------------------------------- | ------------------------------------------------------------------ |
| **mute**   | 无说话：**保留时长**，做静音处理（音量置 0）       | `[{ start_time, end_time }]` 需静音的范围                          |
| **delete** | 整段删除，不参与输出（仅用于冗余内容，非无说话段） | `[{ start_time, end_time }]` 需删除的范围                          |
| **concat** | 句内剪辑：取多个 trim 拼接，中间删除               | `[{ start_time, end_time }, ...]` 需**保留**的片段列表，按顺序拼接 |
| **keep**   | 原样保留                                           | `[{ start_time, end_time }]` 原片段范围                            |

## 时间一致性

- **concat 与 deleted_parts**：actionTime 的保留区间不得与 deleted_parts 中要删除的区间重叠
- **mute 段**：显式 mute 的时间范围不得与相邻 keep 段重叠

## reason 规范

| 场景            | 格式示例                                                         |
| --------------- | ---------------------------------------------------------------- |
| 开头/片段间静音 | `片段 0-230ms 无说话，音频做静音处理`                            |
| 片段间静音      | `片段间静音 18490ms（约 18.5s），无说话，保留时长做静音处理`     |
| 句内重复        | `句内重复：一个一个（A+A 模式），删前留后，保留最后一个「一个」` |
| 语气词          | `语气词：嗯/啊/呃，按删除边界拼接`                               |
| 卡顿词          | `卡顿词：那个那个，删前留后`                                     |
| 保留            | `保留`                                                           |

## 句内 concat 示例

原文「我最近做的一个一个口播剪辑的工具」，去掉第一个「一个」：

```json
{
  "text": "大家好，今天跟大家聊一下我最近做的一个口播剪辑的工具。",
  "source_text": "大家好，今天跟大家聊一下我最近做的一个一个口播剪辑的工具。",
  "start_time": 230,
  "end_time": 7470,
  "reason": "句内重复：一个一个，删前留后",
  "action": "concat",
  "actionTime": [
    { "start_time": 230, "end_time": 5310 },
    { "start_time": 5950, "end_time": 7470 }
  ]
}
```

`actionTime` 为保留并拼接的源时间范围，中间 5310-5950ms 被删除。

### 删前留后（顺句词/语气词）示例

原文「其实啊，做视频内容最大的痛苦是什么？」，去掉「其实啊」：

```json
{
  "text": "做视频内容最大的痛苦是什么？",
  "source_text": "其实啊，做视频内容最大的痛苦是什么？",
  "start_time": 18980,
  "end_time": 22220,
  "reason": "顺句词：其实啊，直接移除",
  "action": "concat",
  "actionTime": [{ "start_time": 19900, "end_time": 22220 }]
}
```

- `start_time: 18980` 须覆盖整句，含删前 18980–19900
- `actionTime` 仅保留部分 19900–22220
- `deleted_parts` 对应项：`start_time: 18980, end_time: 19900`（延伸至首个保留字前）
