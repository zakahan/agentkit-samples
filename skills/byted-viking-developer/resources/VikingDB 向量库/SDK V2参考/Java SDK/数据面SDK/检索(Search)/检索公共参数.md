检索功能包括多个接口组成，分别对应不同的检索模式和业务场景。
# 接口说明
V2接口则将不同检索模式对应到不同的接口，更加清晰，便于您针对自己的业务场景，选择合适的检索方式。

## filter结构
使用filter的前提是，相应的标量字段设置为了标量字段（设置了scalarIndex）
| 算子 | 适用字段类型 |  | 示例 |
| --- | --- | --- | --- |
| must | string、int64、bool、list<string>、list<int64> | 针对指定字段名生效，语义为必须在 [...] 之中，即 "must in" | ```JSON <br> { <br>   "op": "must",     <br>   "field": "region",  <br>   "conds": ["cn", "sg"] <br> } <br> ``` <br>  |
| must_not <br>  | string、int64、bool、list<string>、list<int64> | 针对指定字段名生效，语义为必须不在 [...] 之中，即 "must not in" <br>  | ```JSON <br> { <br>           "op": "must_not",         <br>           "field": "data_type", <br>           "conds": [1,2,3] <br> } <br> ``` <br>  |
| and | / | 逻辑算子，针对逻辑查询需求，对多个条件取交集 | ```JSON <br> { <br>   "op": "and",         // 算子名 <br>   "conds": [           // 条件列表，支持嵌套逻辑算子和 must/must_not 算子 <br>     { <br>       "op": "must", <br>       "field": "type", <br>       "conds": [1] <br>     }, <br>     { <br>         ...            // 支持>=1的任意数量的条件进行组合 <br>     } <br>   ] <br> } <br> ``` <br>  |
| or | / | 逻辑算子，针对逻辑查询需求，对多个条件取并集 | ```JSON <br> { <br>   "op": "or",         // 算子名 <br>   "conds": [           // 条件列表，支持嵌套逻辑算子和 must/must_not 算子 <br>     { <br>       "op": "must", <br>       "field": "type", <br>       "conds": [1] <br>     }, <br>     { <br>         ...            // 支持>=1的任意数量的条件进行组合 <br>     } <br>   ] <br> } <br> ``` <br>  |
| range | int64、float32 <br>  | 针对指定字段名生效，语义为必须在指定范围内。配置使用`gte`（大于等于）, `gt`（大于）, `lte`（小于等于）, `lt`（小于），用以圈定一维范围。 |  <br> ```Python <br> // price在[100.0, 500.0) <br> { <br>   "op": "range",     <br>   "field": "price", <br>   "gte": 100.0,  <br>   "lt": 500.0 <br> } <br>  <br> //price >= 100.0 <br> { <br>   "op": "range",       <br>   "field": "price", <br>   "gte": 100.0  <br> } <br> ``` <br>  <br>  |
| time_range | date_time  | 时间点筛选，配置使用`gte`（大于等于）, `gt`（大于）, `lte`（小于等于）, `lt`（小于），用以圈定一维范围。 |  <br> ```JSON <br> { <br>     // 检索北京时间 2025-08-12 00:00:00 ~ 2025-08-13 00:00:00 之间的数据 <br>     "op": "time_range",       <br>     "field": "f_date_time", <br>     "gt": "2025-08-12T00:00:00+08:00", <br>     "lt": "2025-08-13T00:00:00+08:00", <br> } <br> ``` <br>  |
| geo_range | geo_point | 地理距离筛选，使用 `center `为中心点，`radius `为距离半径。 <br>  |  <br> ```JSON <br> { <br>     "op": "geo_range",    <br>     "field": "f_geo_point",                  <br>     "center": "116.412138,39.914912",        <br>     "radius": "10000m"                 <br> } <br> ``` <br>  |

## 后置处理结构
| 算子 | 适用字段类型 | 介绍 | 示例 |
| --- | --- | --- | --- |
| score_fusion | string | 分数融合算子，用于融合指定的时空标量的分数。 | ```JSON <br> { <br>         "op": "score_fusion", <br>         "fusion_by": "add", <br>         "addition_score_weight": 0.6,  //附加score的权重为0.6   ann score的权重为1-0.6=0.4 <br>         "addition_score": [ <br>             { <br>                 "factor": 1.0, <br>                 "base_value_from": "scalar_field", <br>                 "field": "f_sales" <br>             }, <br>             { <br>                 "factor": -10.0, // 投诉多的商品，score应该降低 <br>                 "base_value_from": "scalar_field", <br>                 "field": "f_complain" <br>             } <br>         ] <br> } <br> ``` <br>  <br>  |
| string_contain <br>  | string <br>  | 关键词匹配过滤算子。表示该字段内容包含pattern。 | ```JSON <br> { <br>    "op": "string_contain", <br>    "field": "name", <br>    "pattern": "bar" <br> } <br> ``` <br>  <br>  |
| string_match <br>  | string | 正则匹配过滤算子。 <br>  | ```JSON <br> { <br>    "op": "string_match", <br>    "field": "name", <br>    "pattern": "^[0-9A-Za-z_]+$" <br> } <br> ``` <br>  |
| enum_freq_limiter | string | 频控算子。用于保证一次召回的结果中, 一个特定取值出现的总数不超过 `threshold` 次。 | ```JSON <br> { <br>   "op": "enum_freq_limiter", <br>   "field": "city" <br>   "threshold": 5 <br> } <br> ``` <br>  |
